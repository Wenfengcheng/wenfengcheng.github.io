Xamarin 中能否使用原生 SDK

从头开始还是绑定？

使用Xamarin，在编写SDK时通常有两种选择。您可以从理想情况下开始使用足够通用的PCL项目来定位所有平台，或者您可以创建一个绑定项目，该项目基本上将本机库“包装”在C＃API中。

如果你的SDK不能访问任何原生功能（或者很少），你可能更好的只是创建PCL PCL还可以为标准iOS，Android和UWP之外的其他平台提供更大的灵活性。

在我们的例子中，由于我们依赖于许多底层本机API（例如围绕安装跟踪），我们决定采用绑定方法。

Binding 步骤

在 Xamarin 中使用第三方 SDK 的大概步骤：

  1. 使用原生构建 .jar/.aar(用于Java绑定) 和 .a/.framework(用于OC/Swift绑定) 文件。

    友情提示：
    这里可以 clone or copy Java, OC, Swift 代码自己构建，或者直接使用第三方已经生成好的 .jar/.aar 和 .a/.framework

  2. 在 Visual Studio/Visual Studio for Mac 中为 Android 创建一个空的 Binding Library(绑定库) 并添加 .jar/.aar, Mono 框架将尽可能多为我们创建 C＃ 类。
  3. 为 iOS 创建一个 Fat library(完整架构库), 它应该包含 x86 和 arm 体系结构。
  4. 使用 Objective Sharpie(sharpie bind) 为iOS自动生成C＃类。我们可以从 Pod 自动创建 或.h文件 手动生成它们 。此命令将创建两个文件：ApiDefinitions.cs 和 StrucsAndEnums.cs 。
  5. 在 Visual Studio/Visual Studio for Mac 中为 iOS 创建一个空的 Binding Library(绑定库)，并添加我们在 步骤4 中创建的文件。

  6. 重新构建并检查一切正常。如果有一些错误，我们可以直接在 Visual Studio/Visual Studio for Mac 或文档中检查它们。

绑定 Java 库

适用于 Android 的第三方库生态系统是巨大的。 因此，通常最好使用比创建一个新的现有 Android 库。 Xamarin.Android 提供两种方法来使用这些库：
创建绑定库自动包装与库C#包装器，因此可以调用 Java 代码通过C#的调用。
使用Java 本机接口(JNI) 来直接调用 Java 库代码中的调用。 JNI 是一个编程框架，允许调用并由本机应用程序或库调用的 Java 代码。

绑定 OC 库

为 Xamarin 创建 OC 的绑定库大概为三个步骤：

  1. 编写C#”API 定义”来描述如何本机 API 公开在.NET 中，以及它如何映射到基础 Objective-c。 这是使用标准C#构造喜欢interface和各种绑定特性(请参阅此简单示例)。
  2. 一旦您编写”API 定义” C#，其生成”绑定”程序集进行编译。 这可以在命令行也可以使用绑定项目 Visual Studio for Mac 或 Visual Studio 中。
  3. 该”绑定”程序集然后添加到 Xamarin 应用程序项目中，以便可以访问使用你定义的 API 的本机功能。 绑定项目是完全独立于应用程序项目。

注意事项：
可以使用的帮助自动执行步骤 1 目标 Sharpie。 它会检查 Objective C API 并生成建议C#”API 定义”。 您可以自定义目标 Sharpie 创建的文件并将其绑定项目 （或在命令行上） 来创建绑定程序集。 目标 Sharpie 不会创建单独的绑定，而是只是较大过程的一个可选部分。
