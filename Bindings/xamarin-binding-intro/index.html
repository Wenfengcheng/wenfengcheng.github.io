Xamarin 中能否使用原生 SDK

更新于: 2019-03-01 | 分类于 Bindings | | 阅读次数:
字数统计: 1.3k | 阅读时长 4 分钟

写在前面：
对于刚接触到 Xamarin 的朋友来说，了解到使用 Xamarin 可以快速搭建跨平台的移动应用 App, 但是对于能否使用原生开发中常用的第三方SDK或开源组件这个问题，许多人都保持着不确定的态度。这个问题的答案是肯定的，几乎所有在原生开发中被使用到的组件，特性，框架等都可以运用到 Xamarin 开发之中 (但是这其中需要我们将通过原生编写的组件转换为可以被 C# 识别且调用的库)，而且 Xamarin 不光可以使用原生常用的组件库，还可以使用 C#/.Net Standard 提供的庞大组件库。举个例子来说 iOS 和 Android 其在发送网络请求时各自平台使用的肯定时特有的工具库和机制，但是在 Xamarin 中这部分我们可以直接 C# 提供的处理网络请求工具库，这部分在 iOS 和 Android 两个平台上都能正常运行，且映射到不同的平台及为各自平台原生的实现形式。接下来我们就详细说说 Xamarin 中如何使用原生库。

从头开始还是绑定？

大家在准备使用某个原生库之前，可以按照以下几步去定位转换这个库的方法：

  1. 首先你想使用某个原生库，第一时间先上 nuget 上搜一下是否有现成已经转换好的可以直接使用的库。
  2. 如果没有，可以到 github 或者 bing/google 等平台搜索 Xamarin-[Your-Library-Name]，如果有现成的库，可以直接 clone/download 下来添加到自己的项目中使用。

  3. 如果都没有，就需要你去查看这个库的复杂程度以及是否涉及到许多底层原生 API，如果只是简单的 UI 工具库，可以直接对照原生代码，翻译 C# 的代码形式即可使用；如果功能比较复杂，且其中包含一些 C/C++ 库的引用，这里可能就需要通过手动绑定来转换。

Binding 步骤

在 Xamarin 中使用原生第三方组件的时候，大部分都需要通过 Binding(绑定) 的形式来进行转换，少部分可以通过翻译代码来实现，使用 Binding(绑定) 来集成第三方组件大概步骤为：

  1. 使用原生构建 .jar/.aar(用于Java绑定) 和 .a/.framework(用于OC/Swift绑定) 文件。

    友情提示：
    这里可以 clone or copy Java, OC, Swift 代码自己构建，或者直接使用第三方已经生成好的 .jar/.aar 和 .a/.framework

  2. 在 Visual Studio/Visual Studio for Mac 中为 Android 创建一个空的 Binding Library(绑定库) 并添加 .jar/.aar, Mono 框架将尽可能多为我们创建 C＃ 类。
  3. 为 iOS 创建一个 Fat library(完整架构库), 它应该包含 x86 和 arm 体系结构。
  4. 使用 Objective Sharpie(sharpie bind) 为iOS自动生成C＃类。我们可以从 Pod 自动创建 或.h文件 手动生成它们 。此命令将创建两个文件：ApiDefinitions.cs 和 StrucsAndEnums.cs 。
  5. 在 Visual Studio/Visual Studio for Mac 中为 iOS 创建一个空的 Binding Library(绑定库)，并添加我们在 步骤4 中创建的文件。

  6. 重新构建并检查一切正常。如果有一些错误，我们可以直接在 Visual Studio/Visual Studio for Mac 或文档中检查它们。

绑定 Java 库

适用于 Android 的第三方库生态系统是巨大的，因此，使用现成的 Android 第三方库比重新去新建一个同样功能库要更好。 Xamarin.Android 提供两种方法来使用这些库：

  1. 创建 Bindings Library(绑定库), 其中将自动包含这个原生库以及该原生库的 C# 包装定义，因此可以通过 C# 的调用 Java 代码。

  2. 使用 Java Native Interface (JNI) 来直接调用原生 Java 库代码。 JNI 是一个编程框架，允许 Java 调用并且支持由本机应用程序或库调用的 Java 代码。

绑定 OC 库

为 Xamarin 创建 OC 的绑定库大概为三个步骤：

  1. 编写描述如何将原生 API 暴露在.NET 中 C# API 定义，以及它如何映射到 Objective-C。 这可以通过使用标准C#构造 (如interface) 和各种绑定特性实现。
  2. 一旦你编写完了原生 API 在.NET 中 C# API 定义 的实现，其编译生成 binding assembly(绑定程序集)。 实现方式可以通过 命令行 也可以使用 绑定项目 Visual Studio for Mac 或 Visual Studio 中。
  3. 之后添加 binding assembly(绑定程序集) 到 Xamarin 应用程序项目中，以便你通过自定义的 C# API 定义 访问原生组件的功能。 绑定项目是完全独立于应用程序项目。

注意事项：
步骤 1 可以通过 Objective Sharpie 自动完成。 它会检查 Objective-C API 并生成建议 C# API 定义。 您可以自定义 Objective Sharpie 自定生成的文件并将其绑定项目(或在命令行上)来创建绑定程序集。Objective Sharpie 不会创建单独的绑定，而是整个绑定过程的一个可选部分。详细的绑定工作原理可以参考 How Binding Works。
